# 실시간 커머스 플랫폼

## 구현 내용

Java 콘솔 기반 커머스 플랫폼을 객체 지향 설계 원칙을 적용하여 구현했습니다.

## 1. 도메인 설계

각 클래스의 역할과 책임을 분리하여 설계했습니다.

- `Product` : 상품명, 가격(`long`), 설명, 재고수량 관리
- `Category` : 카테고리명과 `List<Product>` 관리. `CommerceSystem`이 상품을 직접 관리하는 것이 아닌 Category가 소유하도록 책임 분리
- `Customer` : 고객명, 이메일, 등급(`Grade`) 관리
- `Cart` : 장바구니 항목(상품명, 수량, 가격) 관리. 담을 당시 가격을 스냅샷으로 저장

## 2. 서비스 설계

`CommerceSystem`이 모든 콘솔 입출력과 흐름 제어를 담당하도록 설계했습니다.

- 도메인 객체(Product, Category)는 데이터 보관만 담당
- 출력/입력/분기는 `CommerceSystem`이 전담
- 메서드 단위로 역할 분리 (`printCategory`, `enterCategory`, `printProducts`, `printCart` 등)

## 3. 장바구니 및 주문 기능

- 상품 선택 시 장바구니 추가 여부 확인
- 재고 확인 후 부족 시 경고 메시지 출력
- 동일 상품 담을 시 수량 증가 (재고 초과 방지)
- 장바구니에 상품이 있을 때만 주문 관리 메뉴 출력
- 주문 취소 시 장바구니 초기화

## 4. 예외 처리

`InputHandle` 유틸 클래스로 입력 예외를 한 곳에서 관리합니다.

- 숫자 외 입력 시 `InputMismatchException` 처리
- 범위 벗어난 번호 입력 시 재입력 요청

## 어려웠던 점

`CommerceSystem`이 `List<Product>`를 직접 관리하던 구조에서 `Category`가 상품 목록을 소유하도록 책임을 분리하는 과정이 어려웠습니다. 단순히 클래스를 나누는 것이 아니라 **누가 데이터를 소유하고, 누가 그 데이터를 사용하는지** 명확히 구분하는 것이 중요하다는 것을 배웠습니다.

## 배운 점

객체지향 설계에서 중요한 것은 단순히 클래스를 나누는 것이 아니라 **각 클래스가 하나의 변경 이유만 가지도록(단일 책임 원칙)** 설계하는 것임을 배웠습니다. 또한 `final` 키워드를 통해 의도치 않은 재할당을 막고, `try-with-resources`로 자원을 안전하게 관리하는 방법을 익혔습니다.
